프로젝트 개요 OVERVIEW

미션

  • 생산현장의 실물 도면 사용을 편리하게 지원
  • 업무가 더 늘어나면 안 된다 (자동화 우선)
  • 권한 기반 출력 + 워터마크로 보안 확보
  • 우회 경로(백도어)를 체계적으로 억제

현황 (AS-IS)

  • 도면 서비스가 4곳으로 분산 (PDM, NPDM, ERP, EP-DRIVE)
  • 백도어 3곳 — 카톡, EP-DRIVE, 메일로 도면 유출
  • 5개 부서가 도면 배포에 고통
  • 제품기술팀이 병목 허브 (월 40건, 16시간/월 소요)
  • 도면 출력관리 전담 인력 1인 (일 8시간 기준, 도면업무 약 6시간/일)

최초 설계 아키텍처 (PLM 1.5단계)

CAD → PLM → NPDM 경로가 메인 프로세스로 설계됨. IMOS/AutoCAD 기반 도면의 배포 경로가 아키텍처에 존재하지 않음 — PDM 사용을 줄이는것이 목표였으나 imos/autocad 도면의 배포처가 없어 사용이 지속되고 있음

문제: CREO(CAD) 기반 도면은 PLM → NPDM 경로로 배포 가능하지만,
IMOS / AutoCAD 기반 도면은 PDM에만 존재하며 외부 배포 경로가 없음 → 제품기술팀이 수동으로 배포하는 병목 발생
해결: PDM의 역할을 IMOS/AutoCAD 도면의 생성 및 관리 전용으로 축소하고,
도면 서비스(조회/출력/배포)는 NPDM(외부 배포)과 ERP(내부 합본 출력)가 담당
PDM은 도면의 원천 저장소 역할만 유지하고, 사용자 접점은 NPDM(외부)과 ERP(내부)로 이관
→ PDM 권한 보유자 축소로 보안 강화 + 사용자는 익숙한 ERP에서 바로 출력
