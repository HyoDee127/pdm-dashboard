② 경로 변환
cygpath -w 로 git-bash unix 경로를 Windows 경로로 변환. sqlcmd.exe가 Windows 네이티브 프로세스라 필요
③ 쿼리 실행
sqlcmd -u (UCS-2 LE 출력) + -s "|" (파이프 구분자) + -W (공백제거) 로 tmpfile에 저장
④ 인코딩 변환
Python 3로 UTF-16 → UTF-8 재인코딩. sys.stdout.buffer.write(...) 로 cp949 콘솔에서도 한글 정상 출력
⑤ 정리
tmpfile trap 자동 삭제. 비밀번호는 프로세스 인자로 노출되지만 bash history 외엔 영속 없음

★ 데이터 오염

ITEM_ACNT 컬럼 의미 오매핑 (Sybase→MSSQL 전환 실수 추정)

증상: PDM T_ITEM_BOM.ITEM_ACNT 1,097,287건 전부 M03xxx(ERP 자재계정코드)로 채워져있음. 원래 의도는 T_CODE의 X04 계열(X04A=PDF 도면, X04B=DWG, ..., X04H=가공도) 파일타입 코드.
연쇄 영향: ① CtDrawDao.getMissList의 철물/테이프/스티커 자동제외 로직 무력화 ② CtDrawController.mapping.do의 파일타입 분기(X/0/else)에서 M-시작 값이 else로만 매칭되어 모든 주문품이 X03M(부자재)+PDF로 저장 ③ 주문품 등록 시 파일타입 선택 기능 사실상 사라짐.
원인 가설: 전임자가 Sybase→MSSQL 마이그 시 s_mathed.matacnt(과거 파일타입 추정) → TT_MATDTL.MATACNT(자재계정) 컬럼명 유사성만 보고 매핑. 두 컬럼은 이름만 같고 도메인이 완전 다름.
확정 필요: 운영 PDM의 ITEM_ACNT 분포 확인, 현업 인터뷰로 과거 파일타입 선택 UI 유무 확인, ERP팀에 Sybase 시절 matacnt 의미 확인.

C.java의 UPLOAD_PATH = "C:/development/PDM_WEB/WebContent/_upload"로 박혀있음. 운영은 파일서버(UNC/공유경로)에 실파일이 있고, 로컬에는 일부 샘플만 존재. T_DRAW_FILE의 PATH+FILE_NM이 로컬에 없으면 viewer가 "도면파일이 존재하지 않습니다" 반환. 환경별 분기 없음.

2. EP-DRIVE 백도어 — 가장 큰 구조적 리스크

인터뷰로 드러난 실제 운영 모습

사내협력사가 도면을 직접 열람/다운받지 못하는 상황을 커버하기 위해 제품기술팀이 PDM에서 도면을 내려받아 EP-DRIVE에 올리고, 사내협력사가 접속해 사용하는 구조가 일상적으로 운영 중. 편의 차원의 실무 해법이었지만 시스템 관점에서 네 가지 근본 문제 발생:

① 권한 체계 이탈
공식 권한 체계 밖에서 도면이 유통됨. 누구든 EP-DRIVE 접근만 되면 열람 가능
② 유효본 분리
원본(PDM/PLM) ↔ 배포본(EP-DRIVE) 분리. 원본 개정 시 EP-DRIVE 구버전이 현장에 남음
③ 제품기술팀 부담
수동 업로드 · 정리 · 구버전 교체를 지속적으로 떠안음 (단순반복 노동)
④ 이력 추적 불가
누가/언제/무슨 목적으로 도면을 열람·출력·반출했는지 추적 체계 없음
3. 배포 주체별 업무 — 왜, 어떻게 도면을 확보하고 있는가 (AS-IS)

"도면 배포"는 하나의 업무가 아님. 주체별로 목적·경로·고통이 다름. 설계도 주체별로 분리해야 함.

3.1 생산팀 → 협력업체 (재공품 외주견적)

왜
CAPA 초과/생산스펙 이슈로 재공품 외주 필요. 협력업체에 가능 여부/견적 의뢰 위해 도면 전달
어떻게 (현재)
도면배포 요청서 → 제품기술팀 의뢰 → 회신 대기 → 카톡/메일로 1~4개 업체에 개별 발송 (워터마크는 무료 프로그램)
고통
BOM 수만큼 조회/다운 반복. 도면 개정 여부 확신 불가(IMOS는 리비전 없이 덮어쓰기). 오류 책임 우려 → 제품기술팀 의뢰로 일원화 → 병목 심화
NPDM 배포
미사용. 카톡/메일이 실질 채널 (백도어)

3.2 구매팀 → 협력업체 (자재 외주견적)

왜
신규/단종자재의 생산 가능 여부, 단가 산정, 업체 선정 비교견적
어떻게 (현재)
제품기술팀 요청 or PLM 직접 조회. 업체에 메일/카톡 발송. 워터마크 없음
고통
BOM 수만큼 반복 다운. 개정 확신 불가(같은 이유). 오래된 자재/목재 조회 불가. 연구소 직접 발송은 통제 밖
NPDM 배포
미사용. 카톡/메일이 실질 채널 (백도어)

3.3 품질보증팀 → 사내협력사 (현장 품질검사)

왜
사후 하자 검증(출고 후 품질검사 여부/포인트 확인) + 현장 검사 활성화(실물 도면 기반 품질체크)
어떻게 (현재)
PDM/NPDM/EP-DRIVE 다채널에서 도면 수집 + ERP 부품규격표 → 매칭 바인딩 → 현장 배포 → 검사기록 수기 → 회수 → 3개월 보관. 별도 인력 1인 고용, 일 11시간 작업
고통
IMOS 도면 전후면 표기 불가(검사 치명적). PDM인지 NPDM인지 EP-DRIVE인지 하나씩 확인 필요. 바인딩 인력을 검사원으로 돌리고 싶음

3.4 제품기술팀 — ★ 병목 허브 (현장 + 외주 이중 부담)

왜 집중됐나
① BOM 수만큼 반복 다운 ② 개정 진본 보장(다른 팀이 다운하면 구버전 유통 위험) → 오류 책임 우려로 모든 팀이 제품기술팀에 일원화 의뢰
병목 A: 현장 배포
품질/사내협력사의 현장용 요청 → BOM별 PDM/PLM/ERP 오가며 다운 → 바인딩 → EP-DRIVE 수동 업로드(사내협력사용) → 구버전 교체
병목 B: 외주 배포
생산(재공품 견적) + 구매(자재 견적) + 자체 상품 견적 → BOM별 다운 → 수동 워터마크 → 카톡/메일 발송. NPDM 외주배포 기능 미사용
결과
전사 병목(배포 주체 4팀: 생산/구매/품질/제품기술 요청이 전부 여기로) + 본연의 설계 검토 밀림 + 휴먼에러(최신본 누락/잘못된 도면 배포)
NPDM 배포
미사용. 전 배포가 비공식 채널(카톡/메일/EP-DRIVE)로 수행됨

13. 부서별 내러티브 요약

제품기술팀 — 현장+외주 이중 병목, NPDM 미활용

현장 배포(병목 A): 품질/사내협력사의 현장용 요청 접수 → BOM별 PDM/PLM/ERP 오가며 다운 → 바인딩 → EP-DRIVE 업로드 → 구버전 교체.
외주 배포(병목 B): 생산(재공품) + 구매(자재) + 자체 상품견적 → BOM 수만큼 반복 다운 → 무료 프로그램 워터마크 → 카톡/메일 발송.
이 모든 과정에서 NPDM의 외주배포 기능(업체 이메일 지정 → 해당 품목만 열람 허용)은 단 한 번도 사용되지 않았다. 결과: 설계 검토 밀림 + 단순반복 노동 잠식 + 전원 비공식 채널 의존.

생산팀 — 재공품 외주견적, 카톡이 공식 채널

CAPA 초과/생산스펙 이슈로 재공품을 외주로 돌려야 할 때, 도면배포 요청서를 제품기술팀에 제출하고 회신을 기다린다. 받은 도면을 1~4개 협력업체에 카톡/메일로 개별 발송. 반제품/양산품목은 도면 자체를 받지 못하는 경우도 있다. 임가공 내외작 전환이 수시 발생해 BOM 전체가 필요한데, 수집 과정이 병목에 걸린다. 업체 회신/발송 이력은 카톡 채팅방에만 존재. NPDM에서 업체에 열람 권한을 부여하는 기능을 사용하면 이 과정 전체가 불필요해지지만, 기능 존재 자체를 모르고 있었다.

구매팀 — 자재 견적, 보안 없는 도면 유통

신규/단종자재의 업체 선정을 위해 비교견적을 받아야 한다. 신규자재는 제품기술팀에 요청, 기존자재는 PLM에서 직접 조회하기도 한다. 도면을 메일/카톡으로 업체에 뿌리는데 워터마크가 없다. 어느 업체에 어떤 도면이 갔는지 이력도 없다. 연구소에서 업체에 직접 도면을 보내는 경우도 있어 통제가 안 된다. 오래된 자재/목재부품은 조회 자체가 안 되는 경우도 많다. 브랜드(일룸/퍼시스)별로 업체 선정 경로도 다르다. NPDM 배포 기능 사용 시 권한 통제 + 이력 + 워터마크가 한 번에 해결되지만, 역시 미사용.

품질보증팀 — 실물 도면 수집/보관, 디지털 전환 대상

사내협력사에 실물 도면을 배포해 현장 품질검사 표기를 하게 한다. 검사 완료 도면을 회수해 3개월 보관 후 폐기. ERP 부품규격표 + PDM/NPDM 도면을 각각 뽑아 바인딩하는 작업이 선행. IMOS 도면은 전후면 표기가 안 돼 검사에 사실상 사용 불가. 부품규격표+도면 통합 출력, 이동카드에 재공코드 노출 등을 원하고 있다. 외주업체 직접 전달은 아니라 NPDM 배포 기능과 직접 연결은 안 되지만, 현장 출력 체계(트랙 A)로 바인딩 수작업 제거가 가장 큰 개선 포인트.

사내협력사 (생산현장) — 다채널 도면 수집 + 바인딩 노가다

품질검사 체크를 위해 도면이 필요하지만 접근 경로가 분산돼 있다. ERP 다운로드 차단 → PDM에 별도 접속해 다운 / NPDM은 접속 권한 없어 EP-DRIVE를 통해 다운 — PDM인지 NPDM인지 EP-DRIVE인지 하나하나 다 해봐야 한다. 다운받은 도면과 ERP 부품규격표를 매칭하여 바인딩 후 작업자별 인계. 검사기록은 도면 위에 수기 작성. 이 업무만을 위한 별도 인력 1인 고용, 일 11시간 근무. 바인딩 인력을 검사원으로 전환하고 싶다는 것이 현장의 간절한 요청.

이 발견이 프로젝트에 미치는 영향

  • 트랙 B는 NPDM 기능 활성화로 방향 전환 완료 (04-17). 신규 ERP 견적요청시스템 전면 설계는 불필요. NPDM 기능 검증 + UX 격차 보완이 우선.
  • 카톡/메일 전달은 "편의에 의한 보안 우회"로 재정의. 인지 못 한 백도어. 보안 감사 보고 시 결정적 근거.
  • "왜 안 썼는가"를 파고들면 진짜 UX 이슈 드러남. NPDM 기능이 카톡 대비 불편하면, TO-BE에서 그 UX 격차를 메워야 현업이 전환함.
  • EP-DRIVE 축소 로드맵에 NPDM 연결. EP-DRIVE 대체 = NPDM 배포 기능 활성화 + UX 개선.

다음 액션: NPDM 배포 기능 직접 테스트 > 카톡 대비 UX 격차 분석 > 현업에 기능 존재 안내 > 수용도 측정

  • 현장에서 도면 확인할 때 주로 PC/프린터/태블릿/휴대폰 어디서 보세요?

B. 품질 기록 / 도면 관리 (8분)

  • 출력된 도면 위에 품질검사 기록 적는 방식 — 한 도면에 몇 명이 기록을 남기나요?
  • 기록 적힌 도면은 어디에 보관? 누가 수거?
  • 도면 개정(Rev) 올라가면 어떻게 알게 되세요? 구버전 계속 쓰는 경우 있나요?
  • 같은 도면을 여러 번 출력해야 하는 상황? (찢어짐/더러워짐/분실)

C. 우회 루트 (5분)

  • EP-Drive 외에 도면 받는 경로가 있나요? (카톡/메일/USB/종이 사본)
  • 협력사 간에 도면 공유하는 경우? (우리도 같은 거 만드는데 그 도면 좀…)

D. 개선 기대 (10분) ★

  • ERP에서 생산계획 → 바로 도면 출력 버튼이 있다면 어떠세요?
  • 도면이 파일로는 안 주어지고 화면에서 볼 수 있고 출력만 된다면 작업에 지장 있을까요?
  • 모바일/태블릿으로 도면 확인이 가능하면 쓰시겠어요? 종이 출력 대체 가능?
  • 품질검사 기록을 디지털로 (태블릿에서) 할 수 있다면 받아들여질까요?

E. 보안 수용도 (4분)

  • 출력물에 본인 이름/시간 워터마크 찍히면 저항감?
  • 출력 사유를 1클릭(예: "생산용/품질검사용")으로 고르는 것 괜찮으세요?
  • 회사 지급 단말(PC/태블릿)에서만 출력 가능하다면 어떠세요?

꼭 확보할 것

  • □ 도면 한 건 요청→확보 평균 시간
  • □ 월 도면 대기로 인한 작업 지연 횟수
  • □ 사용 단말 (PC/프린터 위치/태블릿 유무)
  • □ 모바일/태블릿 수용 가능 여부
  • □ 품질기록 디지털화 거부감 수준
팀 3. 구매/생산팀

45분 · 외주 견적/발주 흐름. 검색 UX 핵심

팀 특성: 외주업체에 발주/견적 의뢰 시 도면 필요. PDM은 도면번호 기준 검색이라 제품/자재코드로 찾지 못해 제품기술팀에 의뢰. 외주업체와의 도면 전달도 여기 고유 이슈.

A. 외주 도면 유통 업무 (10분)

  • 주당 몇 건의 외주 견적 요청이 나가고, 그 중 도면이 필요한 건은 몇 건?
  • 보통 어떤 정보로 도면을 찾으세요? (제품코드/자재코드/도면번호/품명)
  • 제품기술팀에 의뢰해서 도면 받기까지 평균 얼마?
  • 외주업체에 도면 전달 방식? (이메일/EP-Drive/FTP/USB)
  • 외주업체가 동일 도면을 여러 번 요청하는 경우 어떻게 처리?

B. 검색 고통 (구체 사례, 10분)

  • 최근 일주일 내 "도면 못 찾아서 시간 까먹은" 사례를 하나 상세히 말씀해주세요
  • ERP에서 자재/제품 코드는 있는데 PDM에서 도면 연결이 안 되는 경우 얼마나 자주?
  • 색상별로 다른 도면이 있는 경우 구별은 어떻게 하세요?
  • 구버전 도면으로 견적 나가서 문제된 경험?

C. 외주업체 측 이슈 (8분)

  • 외주업체가 받은 도면을 얼마나 보관하는지 아시나요?
  • 외주업체 도면 유출 의심 사례 있었나요?
  • 외주업체도 PDM에 접근 가능하게 해달라는 요구 있었나요?

D. 개선 기대 (10분) ★

  • 자재코드로 바로 도면 검색 가능하다면 하루 얼마나 단축될 것 같으세요?
  • 외주업체용 제한 조회 계정(특정 자재만 열람+다운로드 차단)이 있다면 어떠세요?
  • 외주업체가 도면을 받지 않고 링크로 "볼 수만" 있는 방식은?

E. 보안 수용도 (2분)

  • 외주업체에 나가는 도면에 업체명/일시 워터마크 찍히는 것 저항감?
  • 외주업체의 다운로드/출력 이력이 IT에 기록되는 것 문제 없나요?

꼭 확보할 것

  • □ 주당 도면 필요 외주 건수
  • □ 검색 실패 사례 1건 이상
  • □ 외주업체 수 + 보안 요구 등급 가능 여부
  • □ 자재코드 검색 UX 중요도 순위
팀 4. 품질보증팀

45분 · 종이 도면 수집/보관. 디지털화 가능성 타진

팀 특성: 현장의 품질검사 기록이 적힌 도면을 수집/3개월 보관. 도면 배포 체인에 종속적. 도면이 출력되어야 일이 시작됨.

A. 현재 업무 (8분)

  • 월간 수집되는 기록 있는 도면 장수는 대략?
  • 수집 → 정리 → 보관 과정 설명 (누가/어디에/얼마나)
  • 3개월 보관 룰의 근거는? (법규/내부규정/관례)
  • 3개월 후 처리 방식 (파기/스캔/외부보관)

B. 기록 가독성/품질 (8분)

  • 출력된 도면의 가독성 문제? (색상/선 굵기/치수 누락)
  • 여러 명이 한 도면에 기록 남길 때 작성자 구분은 어떻게?
  • 도면이 찢어지거나 더러워져서 재출력하는 경우?
  • 구버전 도면으로 검사되어 문제된 사례?

C. 추적성 요구 (8분)

  • 감사/고객대응 때 "이 제품의 품질기록" 추적 빈도?
  • 추적 시 종이 도면에서 정보 찾는 데 평균 시간?
  • 불량 발생 시 어떤 도면 버전으로 만들어졌는지 파악 현재 가능한가요?

D. 디지털화 수용도 (12분) ★

  • 품질검사 기록을 태블릿에서 직접 도면에 필기하는 방식이 가능하다고 가정. 현장에서 받아들여질까요?
  • 종이 보관이 완전히 없어지면 좋은 점 / 우려되는 점?
  • 법적/규제 관점에서 전자기록이 인정되는지 확인해보셨나요?
  • 감사 대응 시 시스템 조회가 더 편할 것 같으세요, 종이 파일이 더 편하실 것 같으세요?

E. 보안 / 접근 권한 (4분)

  • 품질팀은 다운로드/출력 권한 필요하세요? 아니면 보기만?
  • 외주업체/협력사에게 품질기록 공유 요구가 있나요?

꼭 확보할 것

  • □ 월 수집 장수
  • □ 3개월 보관 규정 근거
  • □ 태블릿 디지털 기록 수용 여부
  • □ 법규 제약 확인 필요 여부
팀 5. IT센터 / 보안 담당 (본사 또는 충주 IT 지원)

45분 · 보안 딜레마 해소 + 운영 부담 현실 파악

팀 특성: 보안 이유로 ERP 도면 다운로드 차단 → 실제로는 EP-Drive로 전량 유통되는 역설. 차단 정책 유지 부담 vs 열고 통제로 전환 시 도전.

A. 현재 보안 정책 운영 (8분)

  • ERP 도면 차단 정책은 언제 도입됐고 도입 배경 사건이 있었나요?
  • 현재 EP-Drive를 통해 도면이 유통되고 있음 — 인지하고 있었나요? 대응은?
  • EP-Drive 업로드/다운로드 이력은 수집되나요? 감사 가능한 수준?
  • 도면 유출 의심/확정 사건이 실제로 있었나요? 처리 결과?

B. 보안 솔루션 경험 (8분)

  • 퍼시스 내에 워터마크 솔루션 사용 중인 것 있나요? (DRM/SecuDoc 등)
  • 프린트 이력 수집 솔루션 운영 중?
  • 문서 보안 제품 도입 검토한 경험 있으시면 결과는?
  • PDM의 출력 이력을 별도 시스템/SIEM에 보내는 요구?

C. 이번 개선 방향 수용성 (12분) ★

  • "파일 다운로드 차단 + 인쇄만 허용 + 워터마크 + 이력" 방향 어떠세요?
  • 인쇄도 PDF로 저장되면 화면 캡처처럼 유출 가능한데, 이 리스크 수용 가능?
  • 이력 보존 기간 권장? (법적/내부규정 기준)
  • 감사 대응 시 이력 조회 인터페이스 요구사항?
  • 협력사(사내) 계정 발급/말소 프로세스 현황? 개선 가능성?

D. 컴플라이언스 / 감사 (10분)

  • ISO/K-ISMS/개인정보/영업비밀 등 관련 규제 중 도면에 적용되는 것?
  • 외부 감사(고객/인증기관)에서 도면 관리에 대한 지적받은 경험?
  • 현재 정책 변경 시 사전 검토/승인 경로?

E. 운영 부담 (4분)

  • 현 차단 정책 예외 요청(승인 필요)이 월 몇 건?
  • 이 개선이 IT 운영 부담을 늘릴까 줄일까? 전망?

꼭 확보할 것

  • □ 과거 유출 사건 유무 및 처리
  • □ 워터마크 기술 기존 운영 유무
  • □ 이력 보존 기간 권장
  • □ 법/규제 제약 체크
  • □ 정책 변경 승인 경로
